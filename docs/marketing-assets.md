# Marketing & Ops Asset Briefs (Canva / Slack / GitHub)

Ready-to-execute briefs. All assets use the brand tokens from
`docs/brand-guidelines.md` (Plaster #F2EEE8 · Espresso #221C16 · Brass #A8895B;
Marcellus display / Karla body / IBM Plex Mono spec).

## Canva set
- **Business card (90×54 mm)** — Front: monogram centered on espresso, brass inner square. Back: plaster; name in Marcellus 12pt, role + phone + email in Plex Mono 6.5pt set as a title block in the lower-right corner.
- **Brochure (A4 tri-fold)** — Cover: full-bleed living-room image, wordmark bottom-left, sheet no. "K-2026" top-right. Inside: process phases as five ledger rows; back panel: pricing table + QR to /contact.
- **Instagram post (1080×1350)** — One image, 24 px plaster border, project name in Marcellus bottom-left, spec strip (location · area · year) in Plex Mono along the bottom edge.
- **Instagram story (1080×1920)** — Full-bleed detail shot; brass hairline frame inset 40 px; single line of copy max.
- **Facebook / LinkedIn banners** — Espresso field, wordmark left third, tagline in Plex Mono, one 4:5 project image right-aligned and bleeding off the top edge.
- **Presentation (16:9, 12 slides)** — Cover / thesis / 5 process slides / 3 case studies / pricing / contact. Plaster slides with espresso section dividers; every slide carries a sheet number.
- **Flyer (A5)** & **Roll-up (850×2000 mm)** — Single hero image ≥60% of area, one claim ("Calmer in year ten."), consultation CTA + QR.

## Slack workspace scaffold
Channels: `#sprint-planning` `#dev-updates` `#design-reviews` `#releases` `#site-photolog`.
- **Sprint planning template** — Goal / Scope (page-level) / Out of scope / Risks / Demo date.
- **Daily update format** — Yesterday · Today · Blocked (3 lines, thread replies only).
- **Release note format** — `v1.x — date` / Added / Changed / Fixed / Lighthouse scores screenshot.
- **Roadmap pin** — M1 core pages → M2 projects+journal → M3 SEO/perf pass → M4 owned imagery & video → Launch.
